## 2. いま動いている仕組み（現状）

### 構成は2つ
1. **スタッフFAQ（鍵なし・別入口）** = `スタッフFAQ.html`
   - LINE風チャットで質問 → AIが社内ルールに沿って回答。
   - 回答に使える知識が無ければ「未回答リスト」に記録。
2. **FAQ管理（ダッシュボードのログイン内側）** = `FAQ管理.html` + `faq-admin.js`
   - 知識ベースの追加・編集・削除、未回答への回答→知識化、資料からのAI一括取込。

### AI呼び出し
- モデル: **`claude-haiku-4-5`**
- 経路: フロント → **GASプロキシ（`GAS_URL`）** → Anthropic API。APIキーはGAS側で秘匿。
- CORSプリフライト回避のため `Content-Type: text/plain` でPOST。
- GAS_URLが空ならブラウザから `api.anthropic.com` 直叩き（claude.ai上の動作確認用フォールバック）。
- API失敗時は知識ベースの**ローカル文字列検索**で代替回答。

### 知識データの構造（1件）
```js
{ id, q: "質問文", a: "回答本文（【知識】【対応手順】【ベスト提案】の見出し可）",
  category: "カテゴリ", source: "出典", images: ["url", ...], addedAt: "YYYY-MM-DD" }
```

### 保存場所（現状）
- 知識ベース／未回答リスト = **localStorage**（キー `miwa.faq.kb.v1`）。初回は `FAQ_KB_SEED` を複製。
- FAQログ = localStorage（キー `miwa.faq.log.v1`）。
- → **全店共有はまだ未対応**。スプレッドシート化は「第2段階」、全端末ログ共有は「第3段階」として設計コメントだけ入っている状態。

## 3. ⚠️ 引き継ぎ時の重要な注意（不足しているもの）

1. **AI中継用のGASは稼働済み（AIはそのまま動く）。ただしソースコードはこのフォルダに含まれていません。**
   `スタッフFAQ.html` / `faq-admin.js` の `GAS_URL`（下記）に、Anthropicへ中継しAPIキーを隠すGASが**デプロイ済み**。よってAI回答は現状のままで動作する。
   ```
   GAS_URL = https://script.google.com/macros/s/AKfycbwgjgqVJNFNnNwNyzc8DsskESrfvoSSTgpK6T2twFPTVyDrhnR2NhNy_CLiajfB1pC_OA/exec
   ```
   - 注意①: その中継GASの `Code.gs` 自体はプロジェクト内に無い（**中身を見たり直したりはできない**）。プロキシ側のロジック変更が必要になったら、GASエディタ側の編集が必要。
   - 注意②: 同梱の `gas_みわ共有API.gs` は**別物（共有データ・認証・Drive用）**。AI中継とは別デプロイ。
   - API失敗時はローカル文字列検索で代替回答するフォールバックも入っている。
2. **`miwa_avatar.png` が無い**（`スタッフFAQ.html` が参照）。アバター・スプラッシュ画像が欠けるだけで動作には影響なし。差し替え可。
3. GAS_URL・スプレッドシートIDは実環境の値が直書きされています。流用時は自分の環境のIDに差し替えること。

## 4. 今後の変更点（次のチャットで実装したいこと）

### ① 料金問い合わせを「料金表スプレッドシート」化
- 各知識から料金を参照するのではなく、**更新しやすい料金表スプレッドシート**を正にする。
- **ブランド別**に料金表を作る：**青のみわ / 緑のみわ**。
- 料金カテゴリ：**クリーニング料金 / 加工料金（クリーニングに別途必要なオプション）/ 特殊品 / 物販**。
- **特殊品は㎡計算あり**：質問で「〇m × ●m」と伝えると平米数と料金を計算して返す。
- **料金表スプレッドシートを取り込む機能**が必要。
- **各知識に書かれている料金は表示しない設定**が必要（料金は料金表に一本化）。

### ② 写真・PDFは Google ドライブ参照に
- 回答に画像が要る時、**Driveのファイルを参照**して表示。
- **対象ワード → ファイル紐付け**で出し分け。
  - 例: 質問「スカートのタグ付け位置を教えて」→ 対象ワード「タグ付け位置／タグ付け+位置／タグ付け+場所」→ Drive内 `タグ付け位置.pdf` を表示。
- 知識ベース追加とは**別に**、ファイル紐付けを追加する機能：
  - **配信日 / 対象ワード（複数可）/ 除外ワード（複数可）/ 対象ファイル（PDF・画像）**。
- 懸念: **Drive接続のセキュリティ**（公開範囲・リンク共有の扱い）を要検討。
  - 補足: 同梱GASの `uploadFile_` は「リンクを知っている全員が閲覧可」でDrive保存している。社外秘資料では公開範囲の設計を見直す必要あり。

### ③ 文章量の多いファイル（就業規則など）の取り込み
- 現状エラー: 「仕分けに失敗しました / Unexpected token '`', "```json [ "... is not valid JSON」。
  - **原因**: AIが ```` ```json ```` のコードフェンス付きで返すとJSON.parseが落ちる。→ パース前にコードフェンス除去＋最初の `[` 〜最後の `]` 抽出を強化すれば回避できる（軽微修正）。
- なるべく**APIトークン消費を抑えて**取り込みたい。
- **現状の取込文字数の上限**を明確にしたい（貼り付けが文字数オーバーしていないか）。
  - 補足: 一括取込は `max_tokens: 2500`、出力JSONがこの上限を超えると途中で切れてパース失敗する。長文は**分割取込**かチャンク化が現実的。
- **ファイルごと知識として保持して回答**する方式（就業規則・大きすぎる文章・自社/参考HPなど）が向くケースがあるか検討。

### ④ 「ついで読み」＝関連回答サジェスト
- AI回答の後に「近い内容で他の回答もあります」と**関連知識を提示**し、やり取り回数を減らす。
- 現状、回答までに**10〜20秒**かかる。立ち作業中の利用が多いので**回答スピード向上**が希望。
  - ただしAPIクラスを上げる（＝コスト増）には慎重。
  - 補足候補: 速度はモデル/トークン調整・ストリーミング表示・ローカル検索の先出しなどで体感改善の余地あり。
