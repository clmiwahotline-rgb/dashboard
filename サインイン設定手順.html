Setup Guide

サインイン設定手順

Google アカウントでのログインを有効にするための、最初の1回だけの設定です。
所要時間：約10分 ／ 作業する人：管理者（Googleアカウントが必要）
💡
この設定では 「クライアント ID」 という文字列を1つ発行し、auth-config.js に貼るだけです。難しい知識は要りません。下の手順どおり進めてください。
1
Google Cloud のプロジェクトを作る
  1. https://console.cloud.google.com/ を開く（管理者のGoogleアカウントでログイン）。
  2. 画面上部のプロジェクトを選択→新しいプロジェクト。
  3. 名前を「みわダッシュボード」などにして作成。作成後、そのプロジェクトを選択。
2
同意画面（OAuth）を設定する
  1. 左メニュー APIとサービス → OAuth 同意画面。
  2. User Type は 外部 を選び作成。
  3. アプリ名「みわダッシュボード」、ユーザーサポートメールに管理者のメール、デベロッパー連絡先にも同じメールを入力 → 保存して次へ を最後まで。
  4. 「対象（テストユーザー）」の画面で、ログインする社員のメールを追加するか、公開ステータスを本番環境にします（本番にすると誰でもボタンは押せますが、実際に入れるのは「アカウント管理」で許可した人だけです）。
※「アプリは確認されていません」と出ても、内部利用なので「続行」で問題ありません。
3
クライアント ID を発行する
  1. 左メニュー APIとサービス → 認証情報。
  2. 上の ＋ 認証情報を作成 → OAuth クライアント ID。
  3. アプリケーションの種類：ウェブアプリケーション。
  4. 承認済みの JavaScript 生成元 に、下のアドレスをそのまま追加（末尾スラッシュなし）：
    https://statuesque-beignet-f1be04.netlify.app
    独自ドメインを使う場合は、そのアドレスも同様に追加してください。
  5. 作成 を押すと クライアント ID が表示されます（…apps.googleusercontent.com で終わる文字列）。これをコピー。
4
クライアント ID を貼り付ける
  1. このサイトのファイル auth-config.js を開く。
  2. 先頭の CLIENT_ID: "" の "" の中に、コピーした ID を貼り付ける：
    CLIENT_ID: "1234567890-abcdefg.apps.googleusercontent.com",
  3. 保存して、サイトを再アップロード（Netlify に上げ直す）。
※ GAS_URL はすでに設定済みなので変更不要です。
5
GAS（共有API）を最新にして再デプロイ
ログイン照合は共有API（Apps Script）が行います。最新の みわ共有API.gs を貼り替え、 authorizeAll を1回実行して権限を許可 → 新バージョンで再デプロイしてください。
  • 許可アカウントは、スプレッドシートに自動でできる 「ユーザー」シート で管理されます。
  • 最初の管理者（clmiwa.hotline@gmail.com ／ satoshi.3104service@gmail.com）は自動登録されます。
  • 以降の追加・削除は、ダッシュボードの 「アカウント管理」 ページから行えます。
完了後の動き： サイトを開くと最初にログイン画面が出ます。許可されたGoogleアカウントでサインインすると入室でき、「次回からサインインを省略する」にチェックしておけば、次回からは手動でログアウトするまで自動で入れます。許可していないアカウントは入室できません。
うまくいかない時：「このアプリはブロックされます」→ 同意画面のテストユーザーに自分のメールを追加。ログインボタンが出ない→ JavaScript生成元のアドレスが正しいか確認（末尾スラッシュなし・https）。それでも困ったら担当へご連絡ください。
