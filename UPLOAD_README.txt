 GitHub アップロード手順
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

【重要】この「deploy」フォルダを「丸ごとアップ」すると
deploy/ というサブフォルダができてしまい正しく動きません。
フォルダの「中身だけ」をリポジトリのルートに置いてください。

━━━ ファイル数が多いため「2回に分けて」アップロード ━━━

GitHub Web は1回100ファイルまでの制限があります。
以下の手順で2回に分けてください。

────────────────────────────────
【第1回】メインファイル（毎回アップ） ← 約83件
────────────────────────────────

1. GitHubリポジトリを開く
2. 「Add file」→「Upload files」をクリック
3. deploy フォルダを開いて
   「manual_assets フォルダを除いた」ファイルをすべて選択
   
   ▼ 選択方法（Windows）
   Ctrl+A で全選択 → Ctrl+クリック で manual_assets を除外
   
   ▼ 選択方法（Mac）
   Cmd+A で全選択 → Cmd+クリック で manual_assets を除外

4. 選んだファイルをブラウザにドラッグ＆ドロップ
   ※ formsite フォルダはそのままドラッグでOK（中のFAQ.htmlも一緒に入ります）
5. 「Commit changes」で保存（まだ完了しないでOK）

────────────────────────────────
【第2回】画像素材（初回と画面が変わった時だけ） ← 21件
────────────────────────────────

6. 再度「Add file」→「Upload files」をクリック
7. deploy/manual_assets フォルダを丸ごとドラッグ＆ドロップ
8. 「Commit changes」で保存

────────────────────────────────
【もっと楽にするなら → GitHub Desktop を使う】
────────────────────────────────

GitHub Desktop を使えばファイル数制限なし・
フォルダごとドラッグするだけで完了します。

   https://desktop.github.com/

インストール後：
1. リポジトリをクローン
2. deploy フォルダの中身を全部 → クローンしたフォルダにコピー
3. 「Commit to main」→「Push origin」

━━━ アップロード確認 ━━━

   https://clmiwahotline-rgb.github.io/v.html

「v2.14 ✓ アップロード成功」が出ればOK。

━━━ GAS スクリプトについて ━━━

Google Apps Script のファイル（.gs）は
GitHub Pages にアップロード不要です。
gas/ フォルダは deploy フォルダには含まれていません。
プロジェクト内の gas/ フォルダを参照してください。
