# みわダッシュボード v.2 — プロジェクト規約

## シフトページ（シフト.html）

クリーニングみわ全12拠点（店舗9＋八潮工場・東川口工場・ルート）の月間シフトを表示するページ。

### 構成ファイル
| ファイル | 役割 |
|---|---|
| `シフト.html` | ページ本体（CSS含む）。React + Babel をCDN読込 |
| `polished-shift-seed.js` | シフトデータ本体。`window.SHIFT_2026_06` を定義 |
| `polished-shift-io.jsx` | CSV取込 / メモ / localStorage 入出力ユーティリティ |
| `polished-stores.jsx` | ページUI（拠点カード・月間表・変更メモ・カバーバー） |
| `polished-atoms.jsx` | 共通アイコン・`StoreTag`・`storeColor`（拠点カラー） |
| `polished-sidebar.jsx` | 共通サイドバー。シフトは「社内」グループ＝ありがとうカードの上 |

### データ構造（`window.SHIFT_2026_06`）
```js
{
  month: "2026-06",
  stores: [{
    store: "八潮工場",        // 表示名（storeColor のキーと一致させる）
    sheet: "八6",             // 元シート名（参考）
    dates: ["2026-06-01", …], // この拠点の日付（ISO）
    events: { "2026-06-07": ["セール"] },        // 日付→イベント配列
    help:   { "2026-06-07": [{ raw, name, time }] }, // 他店からの応援受入(HELP)
    staff:  [{
      name: "キラン",
      days: 22, hours: 132,   // recomputeData() が再計算
      cells: { "2026-06-01": { raw: "受付9-15", time: { s:9, e:15, text:"9-15" } } }
    }]
  }]
}
```
- **曜日は日付から自動算出**（`wdOf`）。データに曜日を持たせる必要はない。
- **他店応援（dispatch）**＝セル先頭の非数字が他拠点名（漢字・役割語でない）。KPI人数に含めず「→店名」で別表示。役割語（`ROLE_WORDS`：受付/事務/レジ/研修/集配/配送/ルート 等）は応援ではなく業務としてKPIに含める。
- **HELP**＝他店から来た応援。`help` に格納し、KPI人数に加算、月間表最下段「応援(HELP)」行に表示。
- **カバーバー**：9–19時を15分刻みで在籍人数を集計。1名=青/2名=黄/3名+=赤。
- **工場（名前に「工場」）**：「最大N名重複」バッジは出さず、3名以下の日に「ヘルプ確認」を表示。

## 新しいシフトファイルの取り込み方（重要）

ユーザーが新しいシフトファイルを入れたら、**同じ `polished-shift-seed.js` 形式**に変換して上書きする。

### A. Excel（.xlsx）の場合 — `run_script` でパース
xlsxは複数シート（拠点×月）。各シートは「週ブロック」の縦積み構造：
1. `run_script` + `readFileBinary` でzipとして展開（`DecompressionStream('deflate-raw')`）。`xl/workbook.xml`（シート名）, `xl/_rels/workbook.xml.rels`（rId→パス）, `xl/sharedStrings.xml`, `xl/worksheets/*` を読む。
2. 拠点シート名の例：本店=`本6`、新田=`新6`、草加西口=`草6`、モール草加=`ﾓ草6`、蒲生=`蒲6`、西友伊原=`伊6`、東川口=`東6`、東川口2号=`2号6`、マミー安行=`マ安6`、八潮工場=`八6`、東川口工場=`東Y6`、ルート=`ﾙｰﾄ6`。（`仮` 付き＝仮版。確定版を優先、無ければ仮）
3. 各週ブロック：A列が `日付` の行に日付シリアル値（B〜H列）。続く行に `曜日` / `…シフト`（工場の人数ヘッダ＝スキップ）/ `…イベント` / `HELP…` / スタッフ各行。
   - **重要：1スタッフ＝2行（セル結合 A6:A7 等）**。上段の名前行に続くA列が空の行は同一スタッフの下段。**上下段を結合**して1日のセルを作る（例：6/4 上段「集配」＋下段「14-19」→ raw「集配 14-19」, time=14-19）。A列が空の行を読み飛ばすと下段の時間が全欠落する既知バグ。複数時間帯がある場合は raw に連結、time は min開始〜max終了の包絡。
4. **日付シリアル**：`Date.UTC(1899,11,30)+serial*86400000`。
5. **氏名のふりがな除去**：氏名末尾のカナ（`/[\u30A0-\u30FF\uFF66-\uFF9F\u3099\u309A]+$/`）を除去。**ただし除去結果が空になる純カタカナ名（例：キラン）は元の名前を保持する**（`stripped || x`）。← これを怠ると純カナ名が消える既知バグ。
6. **時刻**：`(\d{1,2})(:\d{2})?\s*[-~]\s*(\d{1,2})?` → `{s,e,text}`。終了≤開始なら+12（午後）。
7. **メモ行の除外**：休憩ルール等（`/休憩|◆|●|[①-⑳]|目安|まで|以上|合計|分$|^[0-9]/` か長さ>9）はスタッフ扱いしない。
8. 生成後 `recomputeData()` 相当で `days/hours/dates` を再計算し、`polished-shift-seed.js` に `window.SHIFT_2026_06 = …` として書き出す。

（実装の完全な参照は過去の `run_script` を踏襲。`polished-shift-io.jsx` の `ioParseTime` / `ioNormDate` / `recomputeData` が再利用可能なロジック）

### B. CSV の場合 — アプリ内「CSV取込」ボタン or `importShiftCSV()`
列（ヘッダ名は表記ゆれ自動判定、ヘッダ無しは列順 `拠点,スタッフ,日付,時間,役割,種別`）：

| 列 | 内容 | 例 |
|---|---|---|
| 拠点 | 表示名 | 八潮工場 |
| スタッフ | 氏名 | キラン |
| 日付 | YYYY/M/D・YYYY-MM-DD・M/D | 2026/6/1 |
| 時間 | 開始-終了 | 9-15 |
| 役割・備考 | 任意（受付/事務、または応援先店名） | 受付 |
| 種別 | 勤務 / 応援(=HELP受入) 任意 | 勤務 |

`polished-shift-io.jsx` の `importShiftCSV(text, base)` が既存データにマージし、`recomputeData` で再集計、localStorage `miwa.shift.data.v1` に保存。見本は `SAMPLE_CSV`、UIの「見本CSV」でDL可。
